**File 2: proxy/src/main/java/com/example/crateproxy/CrateProxyDriver.java**

Per D-01 (passthrough stub — no SQL rewriting): delegates all JDBC calls directly to org.postgresql.Driver.

```java
package com.example.crateproxy;

import java.sql.Connection;
import java.sql.Driver;
import java.sql.DriverManager;
import java.sql.DriverPropertyInfo;
import java.sql.SQLException;
import java.util.Properties;
import java.util.logging.Logger;

/**
 * Phase 1 passthrough stub.
 * Per D-01: no SQL rewriting — all calls forwarded directly to pgJDBC.
 * Per D-02: Keycloak will crash on BEGIN; that is acceptable for Phase 1.
 */
public class CrateProxyDriver implements Driver {

    private static final Driver REAL = new org.postgresql.Driver();

    static {
        try {
            DriverManager.registerDriver(new CrateProxyDriver());
        } catch (SQLException e) {
            throw new ExceptionInInitializerError(e);
        }
    }

    @Override
    public Connection connect(String url, Properties info) throws SQLException {
        return REAL.connect(url, info);
    }

    @Override
    public boolean acceptsURL(String url) {
        return url != null && url.startsWith("jdbc:postgresql:");
    }

    @Override
    public DriverPropertyInfo[] getPropertyInfo(String url, Properties info) throws SQLException {
        return REAL.getPropertyInfo(url, info);
    }

    @Override
    public int getMajorVersion() { return 1; }

    @Override
    public int getMinorVersion() { return 0; }

    @Override
    public boolean jdbcCompliant() { return false; }

    @Override
    public Logger getParentLogger() {
        return Logger.getLogger("com.example.crateproxy");
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Docker network | docker-compose.yml exposes ports 4200, 5432, 8080 to host |
| Keycloak → CrateDB | JDBC connection over internal Docker network; no TLS |
| Maven builder → proxy-jar volume | Builder writes JAR; Keycloak reads from same volume |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | CrateDB — empty password for `crate` superuser | accept | PoC scope; no production data. Port 5432 bound to host is debug-only. PROD-03 (secrets management) is a v2 requirement. |
| T-01-02 | Spoofing | Keycloak bootstrap admin `admin`/`admin` credentials in compose env vars | accept | PoC scope only. Credentials are plaintext in compose file — PROD-03 deferred to v2. |
| T-01-03 | Tampering | proxy-jar named volume — builder writes JAR, Keycloak reads it | accept | PoC; single-host deployment. In production, volume provenance should be verified. |
| T-01-04 | Tampering | Maven builder fetches deps from Maven Central over HTTPS | mitigate | Maven Central uses HTTPS. maven-cache volume reduces re-download attack surface on subsequent runs. Pin exact versions in pom.xml (42.7.4, 5.3, 3.6.2) to avoid version drift. |
| T-01-05 | Elevation of Privilege | Maven builder runs as root, writes to proxy-jar volume | accept | PoC; single-host. Keycloak UID 1000 can read world-readable files written by root. Mitigated by Docker network isolation. |
| T-01-06 | Information Disclosure | CrateDB admin UI exposed on host port 4200 | accept | Debug port for PoC. Do not expose 4200 in production without authentication. |
| T-01-07 | Denial of Service | `gateway.expected_data_nodes=3` causes cluster to wait indefinitely if a node fails to start | mitigate | `gateway.recover_after_data_nodes=2` allows recovery with quorum of 2 nodes. Compose healthcheck retries=10 with 30s start_period gives 130s for cluster formation. |
</threat_model>
