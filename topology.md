## Layers

### Host
| Port | Protocol  | Description          |
|------|-----------|----------------------|
| 4200 | HTTP/JSON | CrateDB SQL API + UI |
| 5432 | PostgreSQL | CrateDB PG wire      |

### caddy-lb (top-level)
Single entry point exposed to the host. Round-robin load balances across the three mid-tier Caddy instances. Passive health checks remove failed instances automatically.

### caddy / caddy2 / caddy3 (mid-tier)
Each independently load balances across all three CrateDB nodes. Not exposed to the host. Passive health checks remove failed CrateDB nodes automatically.

### cratedb1 / cratedb2 / cratedb3
3-node CrateDB cluster. All nodes are master-eligible вҖ” one is elected active master at a time. Quorum requires 2 of 3 nodes. Tolerates 1 node failure.

## Failure tolerance

| Failed components          | Service status |
|----------------------------|----------------|
| 1 CrateDB node             | вң… Healthy      |
| 2 CrateDB nodes            | вқҢ No master    |
| 1 mid-tier Caddy           | вң… Healthy      |
| 2 mid-tier Caddies         | вң… Healthy      |
| 3 mid-tier Caddies         | вқҢ Unavailable  |
| caddy-lb                   | вқҢ Unavailable  |
